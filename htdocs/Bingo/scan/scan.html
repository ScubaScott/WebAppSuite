OCR Tuning

Name Value How it affects OCR
Chooses how the image is turned into black-and-white. OTSU is automatic, Binary uses a fixed cutoff, and Adaptive adapts to lighting across the cell.
Softens small noise and texture before thresholding. Too much blur can make digits look fuzzy, while too little can leave specks that confuse OCR.
Makes each number larger before OCR. Since cells are now cropped tightly around the ink first, zooming enlarges just the digit rather than the whole cell, so a slightly higher value works well.
Selects the recognition engine. LSTM is usually better for clear digits, while Legacy can be useful with older or unusual setups.
Tells Tesseract how the image is structured. Testing found Single Block most reliable for isolated 1-2 digit bingo numbers. This setting is only used as a last-resort fallback attempt now — the OCR pass tries a few proven combinations first. Avoid Single Char — it can only ever return one character, so any two-digit number gets truncated.
Reverses light and dark pixels. Use this if your card has light numbers on a dark background or if the thresholding is treating the image backwards.
Adds a small cleanup step after thresholding. Dilate and Close can strengthen thin digits, while Erode can remove small specks and noise.
Sets the cutoff for turning grayscale pixels into black or white. Lower values make more pixels white; higher values make more pixels black. Only used when Threshold Mode is Binary.
Controls the size of the neighborhood used for adaptive thresholding. Larger values smooth lighting changes more, which can help on uneven photos. Only used when Threshold Mode is Adaptive.
Adjusts how aggressively adaptive thresholding separates foreground from background. Larger values make the threshold more conservative and can reduce false positives. Only used when Threshold Mode is Adaptive.
Shifts the OCR sampling up or down by one row. The grid is now auto-detected and cropped before slicing, so this should rarely be needed — only use it if the auto-detected grid boundary is slightly off.
Shifts the OCR sampling left or right by one column. The grid is now auto-detected and cropped before slicing, so this should rarely be needed — only use it if the auto-detected grid boundary is slightly off.
On a standard 75-ball card, each column only contains numbers in a known range (B:1-15, I:16-30, N:31-45, G:46-60, O:61-75). When checked, a misread that falls outside its column's range is automatically retried with different crop/zoom/engine settings instead of being accepted as-is. Turn this off if your cards don't follow the standard B-I-N-G-O ranges.
