Chooses how the image is turned into black-and-white. OTSU is automatic, Binary uses a fixed cutoff, and Adaptive adapts to lighting across the cell.
Softens small noise and texture before thresholding. Too much blur can make digits look fuzzy, while too little can leave specks that confuse OCR.
Makes each number larger before OCR. Larger digits are often easier for Tesseract to read, but excessive zoom can also enlarge noise.
Selects the recognition engine. LSTM is usually better for clear digits, while Legacy can be useful with older or unusual setups.
Tells Tesseract how the image is structured. Single Char is often the best choice for isolated bingo numbers because each cell contains just one digit.
Reverses light and dark pixels. Use this if your card has dark numbers on a light background or if the thresholding is treating the image backwards.
Adds a small cleanup step after thresholding. Dilate and Close can strengthen thin digits, while Erode can remove small specks and noise.
Sets the cutoff for turning grayscale pixels into black or white. Lower values make more pixels white; higher values make more pixels black.
Controls the size of the neighborhood used for adaptive thresholding. Larger values smooth lighting changes more, which can help on uneven photos.
Adjusts how aggressively adaptive thresholding separates foreground from background. Larger values make the threshold more conservative and can reduce false positives.
Shifts the OCR sampling up or down by one row. Use this if numbers are being read from the wrong row, such as a value appearing in the free-space position.
Shifts the OCR sampling left or right by one column. Use this if the recognized numbers are landing in the neighboring cell.
